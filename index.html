Q1 What are method parameters in C#?
Syntax: returnType MethodName(type param1, type param2 = default, params T[] rest)
  • Parameters are named inputs in a method signature; arguments are the actual values you pass at call time.
  • Types of parameters:
    • Value (default): copies the argument value.
    • ref / out: pass by reference (modify caller variable / must assign in method).
    • params: accept variable number of arguments as an array.
    • Optional with default values for convenience.
  • Benefits: stronger type-safety, better reuse, and self-documenting APIs.
C# ‚Äî Value, ref, out, params, optional 
using System;

class Calculator
{
    // Value parameters
    public int Add(int a, int b) => a + b;

    // ref parameter (caller must initialize)
    public void Double(ref int x) { x *= 2; }

    // out parameter (method must assign)
    public bool TryDivide(int a, int b, out int result)
    {
        if (b == 0) { result = 0; return false; }
        result = a / b; return true;
    }

    // params (variable arguments)
    public int Sum(params int[] values)
    {
        int total = 0;
        foreach (var v in values) total += v;
        return total;
    }

    // optional parameter with default value
    public decimal PriceWithTax(decimal price, decimal rate = 0.18m) => price * (1 + rate);
}

class Program
{
    static void Main()
    {
        var c = new Calculator();
        Console.WriteLine(c.Add(2, 3));          // 5

        int n = 10; c.Double(ref n);             // n becomes 20
        Console.WriteLine(n);

        if (c.TryDivide(10, 2, out int q))       // q assigned inside method
            Console.WriteLine(q);

        Console.WriteLine(c.Sum(1,2,3,4));       // 10
        Console.WriteLine(c.PriceWithTax(1000)); // 1180
    }
}

Tip: Prefer value parameters unless you actually need ref/out.

Q2 What is a class and an object in C#?
Syntax: class ClassName { /* members */ }  ‚Üí  var obj = new ClassName(...);
  • A class defines a type (data + behavior). An object is a runtime instance of that class.
  • Objects hold state (fields/properties) and expose behavior via methods.
  • Enables encapsulation and reuse.
C# ‚Äî Class vs Object 
using System;

class Car // class (blueprint)
{
    public string Make { get; }
    public string Model { get; }

    public Car(string make, string model)
    {
        Make = make; Model = model;
    }

    public void Honk() => Console.WriteLine($"{Make} {Model} says Beep!");
}

class Program
{
    static void Main()
    {
        Car myCar = new Car("Toyota", "Corolla"); // object (instance)
        myCar.Honk();
    }
}
Q3 What are class members in C#?
  • Members include: fields, properties, methods, constructors, events, indexers, nested types, and destructors.
  • Each member can have access modifiers (public, private, etc.).
C# ‚Äî Common Class Members 
using System;

public class User
{
    // Field
    private string _email;

    // Property
    public string Email
    {
        get => _email;
        set
        {
            if (!value.Contains("@")) throw new ArgumentException("Invalid email");
            _email = value;
        }
    }

    // Auto-property
    public string Name { get; init; } = "Unknown";

    // Method
    public void SendWelcome() => Console.WriteLine($"Welcome, {Name}!");

    // Event
    public event EventHandler? LoggedIn;

    // Indexer
    public string this[int idx] => $"User#{idx}: {Name}";

    // Constructor
    public User(string email, string name) { Email = email; Name = name; }

    // Nested type
    public enum Role { Guest, Member, Admin }
}
Q4 What is a constructor in C#?
Syntax: public ClassName(...) { /* initialize */ }  |  public ClassName() : this(...) (chaining)
  • A constructor runs at object creation to initialize state.
  • Has same name as the class and no return type; can be overloaded.
  • Implicit parameterless constructor exists until you define any constructor.
C# ‚Äî Constructors & Chaining 
using System;

class Point
{
    public int X { get; }
    public int Y { get; }

    // parameterless constructor
    public Point() : this(0, 0) {}

    // parameterized constructor
    public Point(int x, int y)
    {
        X = x; Y = y;
    }

    public override string ToString() => $"({X},{Y})";
}

class Program
{
    static void Main()
    {
        var p1 = new Point();       // (0,0)
        var p2 = new Point(5, 7);   // (5,7)
        Console.WriteLine(p1);
        Console.WriteLine(p2);
    }
}
Q5 What are constructor parameters and how do they save time?
  • Let callers supply required data at creation time ‚Üí object is fully initialized immediately.
  • Reduces extra property assignments and avoids invalid intermediate states.
  • For many parameters, consider object initializers, records, or the Options pattern.
C# ‚Äî Constructor Parameters Save Time 
using System;

class MailSettings
{
    public string Host { get; }
    public int Port { get; }
    public bool UseSsl { get; }

    public MailSettings(string host, int port, bool useSsl)
        => (Host, Port, UseSsl) = (host, port, useSsl);
}

class Program
{
    static void Main()
    {
        var cfg = new MailSettings("smtp.example.com", 587, true);
        Console.WriteLine($"SMTP {cfg.Host}:{cfg.Port} SSL={cfg.UseSsl}");
    }
}

Q1 Types of access modifiers in C#
  • public ‚Äî accessible from anywhere.
  • private ‚Äî only within the containing type (default for members).
  • protected ‚Äî within the type and its subclasses.
  • internal ‚Äî within the same assembly.
  • protected internal ‚Äî either protected or internal.
  • private protected ‚Äî containing class or derived types in the same assembly.
C# ‚Äî Access Modifiers in Practice 
using System;

public class Base
{
    public int A = 1;               // anywhere
    private int B = 2;              // only in Base
    protected int C = 3;            // Base + subclasses
    internal int D = 4;             // same assembly
    protected internal int E = 5;   // protected OR internal
    private protected int F = 6;    // same assembly + subclasses of Base
}

public class Derived : Base
{
    public void Show()
    {
        Console.WriteLine(A); // ok
        // Console.WriteLine(B); // error
        Console.WriteLine(C); // ok
        Console.WriteLine(D); // ok (same assembly)
        Console.WriteLine(E); // ok
        Console.WriteLine(F); // ok (derived + same assembly)
    }
}
Q2 What are properties in C#?
Syntax: type Name { get; set; } | type Name { get; init; } | type Name => expression;
  • Properties provide controlled access to fields; they can validate, compute, or encapsulate storage.
  • Kinds: auto-properties, computed/read-only, init-only, custom get/set with logic.
C# ‚Äî Properties (auto, validation, init) 
using System;

public class Product
{
    // Backing field + validation
    private decimal _price;
    public decimal Price
    {
        get => _price;
        set { if (value < 0) throw new ArgumentOutOfRangeException(); _price = value; }
    }

    // Init-only property
    public string Name { get; init; } = "Unnamed";

    // Computed property (read-only)
    public decimal PriceWithTax => Price * 1.18m;
}

class Program
{
    static void Main()
    {
        var p = new Product { Name = "Keyboard", Price = 1000m };
        Console.WriteLine($"{p.Name}: Rs {p.PriceWithTax}");
    }
}
Q3 What is polymorphism in C#?
  • Compile‚Äëtime polymorphism: method overloading (same name, different signatures).
  • Run‚Äëtime polymorphism: virtual methods + override; base references call derived implementations.
C# ‚Äî Overloading vs Overriding 
using System;

// Compile-time: overloading
class Printer
{
    public void Print(string s) => Console.WriteLine(s);
    public void Print(int n) => Console.WriteLine($"Number: {n}");
}

// Run-time: overriding
abstract class Shape { public abstract double Area(); }

class Circle : Shape
{
    public double R { get; }
    public Circle(double r) => R = r;
    public override double Area() => Math.PI * R * R;
}

class Rectangle : Shape
{
    public double W { get; }
    public double H { get; }
    public Rectangle(double w, double h) => (W,H) = (w,h);
    public override double Area() => W * H;
}

class Program
{
    static void Main()
    {
        var pr = new Printer();
        pr.Print("Hello");
        pr.Print(42);

        Shape s1 = new Circle(3);
        Shape s2 = new Rectangle(4, 5);
        Console.WriteLine(s1.Area()); // circle logic
        Console.WriteLine(s2.Area()); // rectangle logic
    }
}
Q4 What is abstraction in C#?
  • Expose essential behavior while hiding implementation details.
  • Achieved via interfaces and abstract classes.
C# ‚Äî Abstraction with Interface & Abstract Class 
using System;

public interface INotifier
{
    void Notify(string message); // what, not how
}

public abstract class NotifierBase : INotifier
{
    public void Notify(string message)
    {
        var payload = Format(message);
        Send(payload);
    }

    protected abstract string Format(string message); // must implement
    protected abstract void Send(string payload);
}

public class EmailNotifier : NotifierBase
{
    protected override string Format(string message) => $"Subject: Info\nBody: {message}";
    protected override void Send(string payload) => Console.WriteLine($"EMAIL =>\n{payload}");
}

class Program
{
    static void Main()
    {
        INotifier n = new EmailNotifier();
        n.Notify("Order shipped!");
    }
}
Q5 What is an enum in C#?
  • An enum is a named set of constants backed by an integral type (default int).
  • Improves readability and prevents ‚Äúmagic numbers‚Äù.
  • Use [Flags] for bitwise combinations.
C# ‚Äî Enum Basics & [Flags] 
using System;

public enum OrderStatus
{
    Pending = 0,
    Processing = 1,
    Shipped = 2,
    Delivered = 3,
    Cancelled = 4
}

[Flags]
public enum FilePermissions
{
    None = 0,
    Read = 1,
    Write = 2,
    Execute = 4
}

class Program
{
    static void Main()
    {
        OrderStatus status = OrderStatus.Shipped;
        Console.WriteLine(status);

        var perms = FilePermissions.Read | FilePermissions.Write; // combine bits
        Console.WriteLine(perms);
    }
}
